The "Failed to clean up devices" error was caused by a broken database query in the cleanup endpoint. It used raw SQL that Drizzle couldn't run correctly against PostgreSQL.

What was wrong
The bulk-delete query used sql\${kioskDevices.lastSeen} < ${cutoff}``, which threw a server error (500) when you clicked Remove unnamed.

What I fixed
Fixed the cleanup query — uses Drizzle's lt() for a proper timestamp comparison
Matched UI logic — removes unnamed devices offline for 2+ minutes (same as the offline badge)
Added "Remove all" — new button that clears every registered device at once
What you need to do
Redeploy the server so the fix is live on guestflow.aceelectronics.com. Until then, the production site still has the broken code.
