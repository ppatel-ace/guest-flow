###MsGraph##
Graph API calls must go to:  https://graph.microsoft.us/v1.0/users

###How it works###
User opens app on Intune-enrolled device
          ↓
App redirects to:
login.microsoftonline.us/6ab850db-8359-47f8-9e46-ddb57a3f87bd/ oauth2/v2.0/authorize
          ↓
User enters M365 GCC High credentials
          ↓
Conditional Access evaluates:
  - MFA satisfied? ✅
  - Device enrolled in Intune? ✅
  - Device marked compliant? ✅
  (No IP check — device posture is the trust anchor)
          ↓
Token issued with claims:
  given_name, family_name, upn, email
          ↓
Token returned to your Redirect URI
          ↓  (I will need the Reply URL back to the application to pass the token.)
Your app validates token signature against
  https://login.microsoftonline.us/6ab850db-8359-47f8-9e46-ddb57a3f87bd/.well-known/openid-configuration
          ↓
User is authenticated ✅ — session valid for 8 hours

Architecture
All samples use the OAuth2 Authorization Code + PKCE flow.
No client secret or certificate is required for SSO-only use.
User info (name, UPN, email) is read directly from the ID token claims — no Graph API call needed.
User → App → login.microsoftonline.us → MFA + Intune Compliance Check
           ← ID Token (name, UPN, email claims) ←
